# Security Policy

## Supported Versions

We provide security updates for the latest stable version of SysCat.

## Reporting a Vulnerability

If you discover a security vulnerability, please email security@syscat.dev (or open a private security advisory on GitHub).

**Please do not** open a public issue for security vulnerabilities.

## Security Features

- All credentials stored in OS-native secure storage
- No telemetry or data collection
- All code is open source and auditable
- Local-first architecture (no cloud backend)
- Encrypted local database (optional)

## Best Practices

1. Always audit the code before using in production
2. Use least-privilege app registrations in Azure AD
3. Keep SysCat updated to the latest version
4. Review automation rules before enabling auto-fix
5. Export and backup your configuration regularly
